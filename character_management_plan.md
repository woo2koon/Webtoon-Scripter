# 🎨 캐릭터 관리 기능 개선 구현 계획서
> **Webtoon Script Manager (v2.0) - 효율적인 캐릭터 관리를 위한 글로벌 DB 아키텍처**

본 계획서는 웹툰 대사 추출 및 관리 도구에서 회차(Episode) 생성 시마다 캐릭터 정보를 매번 새로 입력해야 하는 현재의 비효율성을 해소하기 위해, **작품(Project) 단위의 통합 캐릭터 데이터베이스(Global Character DB)**를 구축하고 이를 활용할 수 있는 UI/UX 및 데이터 동기화 구현 방안을 다룹니다.

---

## 📌 1. 현재 구조의 문제점 정의

현재 캐릭터 데이터는 각 회차 디렉토리 내에 개별 CSV 파일(`character_info.csv`)로 관리됩니다.

* **극심한 입력 중복**: 새로운 회차를 생성할 때마다 기존 회차에 있던 인물들(예: 주인공 "밤", "쿤")을 매번 똑같이 나이, 성별, 역할을 입력하여 새로 추가해야 합니다.
* **데이터 불일치**: 한 인물의 성격, 나이, 역할 등의 설정이 중간에 바뀔 경우, 이전 회차들과 신규 회차의 데이터가 일관되게 유지되지 못합니다.
* **유지보수 불가**: 전체 회차에 등장하는 총 캐릭터 목록을 한눈에 보거나, 글로벌 수준에서 일괄 편집할 수 있는 공간이 존재하지 않습니다.

---

## 🎯 2. 개선 목표 (To-Be)

1. **작품별 글로벌 캐릭터 풀(Global Character Pool) 구축**
   * 작품 폴더(`PROJECTS_DIR/<Title>`) 직하에 통합 캐릭터 DB 파일(`characters.json`)을 생성하여 관리합니다.
2. **글로벌 DB 기반의 원클릭 로드 및 선택 기능**
   * 회차 생성 시 글로벌 캐릭터 목록에서 체크박스로 바로 선택해 가져오거나 검색하여 바로 카드에 추가할 수 있도록 합니다.
3. **사용자 친화적 캐릭터 속성 확장**
   * 캐릭터별 **대표 색상(Color) 지정** 기능을 추가하여 스프레드시트 편집기 및 뷰어에서 가시성을 획기적으로 향상시킵니다.
4. **기존 데이터와의 완벽한 호환성**
   * 이전 회차의 개별 `character_info.csv` 포맷을 깨뜨리지 않고, 글로벌 DB 정보와 상호 연계(동기화 및 마이그레이션)되도록 설계합니다.

## 🛠 3. 데이터 구조 설계 (Data Architecture)

### 3.1. 글로벌 캐릭터 파일 (`characters.json`)
* **경로**: `PROJECTS_DIR/<Title>/characters.json`
* **포맷**: JSON 형식 (메모, 색상 등 확장 가능한 유연성 확보)

```json
[
  {
    "name": "스물다섯번째 밤",
    "age": "10대",
    "gender": "남성",
    "role": "주연",
    "color": "#3B82F6",
    "memo": "신수 제어 능력이 뛰어난 비선별인원 주인공."
  },
  {
    "name": "쿤 아게로 아그니스",
    "age": "10대",
    "gender": "남성",
    "role": "주연",
    "color": "#10B981",
    "memo": "브레인 역할을 담당하는 쿤 가문의 가주 후보."
  }
]
```

### 3.2. 회차별 캐릭터 파일 (`character_info.csv`)
* **기존 포맷 유지**: `Character, Age, Gender, Role`
* **개선 포인트**: 회차별 파일은 글로벌 DB에서 **'가져온(선택된)' 캐릭터들의 서브셋(Subset)** 역할을 합니다. 글로벌 캐릭터 정보가 업데이트되면 선택 시 최신 정보로 갱신됩니다.

## 🎨 4. UI/UX 개선 계획

사용자가 직관적이고 끊김 없이 캐릭터를 관리할 수 있도록 크게 세 가지 UI 요소를 보완합니다.

### 4.1. 메인 화면: 글로벌 캐릭터 관리 탭 (신설)
* **위치**: 사이드바 혹은 메인 상단 탭에 "캐릭터 관리" 탭을 추가합니다.
* **주요 구성**:
  * **캐릭터 그리드/목록**: 작품 전체의 캐릭터 카드를 카드뷰 형태로 고급스럽게 표시.
  * **캐릭터 추가/편집 다이얼로그**: 이름, 성별, 연령대, 역할 외에 **고유 컬러 피커(Color Picker)** 및 **간단한 메모 입력창** 제공.
  * **일괄 업데이트 버튼**: 글로벌 캐릭터 DB 수정 시, 기존에 생성된 모든 회차 파일(`character_info.csv`)에도 변경 사항을 일괄 전파 및 동기화할 수 있는 버튼 배치.

```
+--------------------------------------------------------------+
| [작품명: 신의 탑] 캐릭터 데이터베이스 관리                  |
+--------------------------------------------------------------+
| [ + 캐릭터 추가 ]                                           |
|                                                              |
|  +-----------------+  +-----------------+  +--------------+  |
|  | 밤  [주연] [남]  |  | 쿤  [주연] [남]  |  | 라헬 [조연]  |  |
|  | 색상: 🔵          |  | 색상: 🟢          |  | 색상: 🟡     |  |
|  | [수정] [삭제]     |  | [수정] [삭제]     |  | [수정] [삭제]|  |
|  +-----------------+  +-----------------+  +--------------+  |
|                                                              |
|                                         [ 🔄 모든 회차 동기화] |
+--------------------------------------------------------------+
```

### 4.2. 회차별 캐릭터 영역: 글로벌 캐릭터 도우미 플로팅 창 (Step 2 - 신설 및 연동)
* **💡 [핵심 제안 반영] 글로벌 캐릭터 도우미 (Floating Character Helper)**:
  * 현재 프로그램에 구현되어 있는 `"관용구 도우미"`와 완전히 동일한 UX 컨셉의 비모달(Non-modal) 플로팅 창을 추가로 제공합니다.
  * 창의 속성은 항상 위에 노출되는 `Qt.WindowStaysOnTopHint`를 적용하여 작업 중 언제든지 곁에 두고 볼 수 있도록 합니다.
  * **더블클릭(Double-Click) 연동**: 플로팅 창에 나열된 글로벌 캐릭터 카드를 더블클릭하면, 해당 캐릭터가 Step 2의 현재 회차 등장 캐릭터 리스트(`CharacterListContainer`)로 즉시 자동 추가됩니다.
  * **드래그 앤 드롭(Drag & Drop) 연동**: 플로팅 창의 캐릭터 카드를 클릭하여 드래그한 뒤, Step 2의 캐릭터 배치 영역에 툭 떨어뜨리면(Drop) `application/x-character-row` 마임 타입을 전송하여 카드가 자연스럽게 신설됩니다.

```
+--------------------------------------------------------------+
| 💡 글로벌 캐릭터 도우미 (항상 위 플로팅 창)                  |
+--------------------------------------------------------------+
| 🔍 [ 캐릭터 이름 검색...                     ]               |
|                                                              |
| * 더블클릭 하거나 캐릭터 목록 영역으로 드래그하여 추가하세요. |
|                                                              |
| +----------------------------------------------------------+ |
| | 👤 밤 [주연] (10대, 남)                 [ 드래그/더블클릭 ] |
| +----------------------------------------------------------+ |
| | 👤 쿤 [주연] (10대, 남)                 [ 드래그/더블클릭 ] |
| +----------------------------------------------------------+ |
| | 👤 라헬 [조연] (10대, 여)               [ 드래그/더블클릭 ] |
| +----------------------------------------------------------+ |
+--------------------------------------------------------------+
```

### 4.3. 대사 배정 스프레드시트 (Step 3)
* **색상 매핑**: 캐릭터 컬럼 콤보박스에서 캐릭터를 선택하면, 대사 테이블(`table_script`) 내 해당 행(Row) 혹은 캐릭터 셀의 배경/글자 색상이 캐릭터 고유 색상으로 은은하게 적용되어 시각적인 피드백을 강화합니다.

## 🚀 5. 단계별 개발 로드맵

### 1단계: 글로벌 캐릭터 데이터베이스 엔진 구현 (1~2일차)
* `config.py` 또는 신규 `db_handler.py` 모듈을 생성하여 `PROJECTS_DIR/<Title>/characters.json` 파일을 조작하는 API 작성.
* `get_global_characters(project_name)`: 작품 내 전체 캐릭터 반환.
* `save_global_characters(project_name, char_list)`: 글로벌 캐릭터 목록 파일 저장.
* 기존 회차 폴더의 `character_info.csv`를 읽어 자동으로 글로벌 DB로 병합 마이그레이션해주는 헬퍼 스크립트 작성.

### 2단계: 글로벌 캐릭터 관리 전용 뷰/다이얼로그 추가 (3~5일차)
* `widgets.py` 내에 `GlobalCharacterManagerTab` 위젯 신설.
* 캐릭터 CRUD(생성·조회·수정·삭제) 기능을 갖춘 미려한 카드 기반의 관리 창 구현.
* 대표 색상을 쉽게 선택할 수 있도록 `QColorDialog`를 내장한 커스텀 컬러 버튼 탑재.

### 3단계: 회차별 등장 캐릭터 선택 및 배정 UI 개편 (6~7일차)
* **글로벌 캐릭터 도우미 플로팅 창 구현**: `widgets.py` 내 `FloatingCharacterViewer` 클래스(QDialog 기반) 생성.
  * 관용구 도우미(`FloatingIdiomViewer`)의 스타일을 계승하여 `Pretendard` 폰트, 세련된 서브 타이틀 및 검색 필터링 폼 적용.
  * 캐릭터 목록 아이템 더블클릭 이벤트 처리: 메인 윈도우의 `add_character_card()` 슬롯과 연동하여 원클릭 등장인물 등록 실현.
  * 드래그앤드롭 이벤트 바인딩: `QListWidget`의 `startDrag()`를 재정의하여 `CharacterRow` 생성용 드래그 데이터 생성 및 전달.
* 메인 화면의 캐릭터 카드 영역 근처에 "글로벌 캐릭터 보기" 토글 버튼 배치 및 연동.

### 4단계: 시각적 디테일 및 동기화 고도화 (8~10일차)
* 대사 배정 스프레드시트(`SpreadsheetTable`)에서 선택된 캐릭터의 고유 색상을 행(Row)에 은은하게 반영 (예: 투명도 15% 적용된 파스텔톤 배경).
* 글로벌 DB에서 캐릭터 설정을 바꿨을 때, 기존의 회차별 폴더의 데이터를 일괄 수정해주는 동기화 엔진 개발.
* 꼼꼼한 예외 처리 및 최종 빌드 테스트.

## 💡 6. 기대 효과

* **작업 속도 300% 향상**: 50화가 넘어가는 장편 웹툰 작업 시, 매 회차마다 등장인물 10~20명을 반복 입력하는 불필요한 행위가 완벽히 소멸됩니다.
* **데이터 관리 신뢰도 확보**: 캐릭터 이름 오타("길동" vs "홍길동") 등으로 인한 엑셀 데이터 추출 오류를 근본적으로 차단합니다.
* **비주얼 생산성 극대화**: 스프레드시트 화면에서 대사 주체를 캐릭터 고유 색상으로 인지할 수 있어, 검수 속도가 매우 빨라집니다.
